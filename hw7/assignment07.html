Part 1: The Gini Index¶

The Gini index measure of impurity for a label distribution

$$ \b{p} = (p_1, \ldots, p_K) $$

in a classification problem with label set $Y = \{1, \ldots, K\}$ was defined as follows in the class notes:

$$ i_{\text{Gini}}(\b{p}) = 1 - \sum_{k\in Y} p_k^2 \;. $$

This index was justified in terms of a certain stochastic decision rule. This part of the assignment reinterprets the Gini index in purely geometric terms.

The $K$-Simplex¶

Since they describe a discrete distribution, the entries of the vector $\b{p}$ are nonnegative and add up to one:

$$ p_k \geq 0 \;\;\;\text{for}\;\;\; k = 1, \ldots, K \;\;\;\text{and}\;\;\; p_1 + \ldots + p_K = 1 \;. $$

The set $P$ of all such vectors with $K$ components is called the $K$-simplex. The equation $p_1 + \ldots p_K = 1$ represents a hyperplane in $\reals^K$, and the inequalities $p_k \geq 0$ together define the first orthant of $\reals^K$. Therefore, the $K$-simplex is a $K-1$-dimensional object, namely, the intersection of a hyperplane with the first orthant.

The figure below shows the $2$-simplex on the left and the $3$-simplex on the right.

The 2-simplex is a line segment, the 3-simplex is an equilateral triangle, the 4-simplex is a regular tetrahedron, and so forth.

The faces of a $K$-simplex are $K-1$ simplices. For instance, the faces of a tetrahedron are equilateral triangles, and each triangle is bounded by line segments.

The centroid of the $K$-simplex is the point

$$ \b{c} = \frac{1}{K} (1, \ldots, 1) \in P $$

(check this on the figure above). As a distribution, it represents the most impure distribution, all of whose entries are equal, with impurity $i_{\text{Gini}}(\b{c}) = 1 - \frac{1}{K}$.

In words, the Gini impurity index of distribution vector $\b{p}$ is equal to the squared radius of the $K$-simplex minus the squared distance of $\b{p}$ from the most impure distribution $\b{c}$. Part 3: Regression Trees¶

A regression tree approximates a function $y = f(\b{x}) : X \subseteq \reals^d \rightarrow \reals$ by a piecewise-constant function $\phi(\b{x})$.

A leaf in a regression tree contains a bounding box in data space, just as a classification tree does, and the boxes for all the leaves together form a partition $P$ of the data space $X$.

Storing the distribution of the training sample values at a leaf is a bit trickier for regressors than it is for classifiers, because the values of the samples do not belong to a categorical set $Y$. Instead, they are arbitrary real values, $y \in \reals$.

One could store a histogram of the values, or perhaps approximate the distribution with a Gaussian, and store its mean and standard deviation.

Instead, for simplicity, we only store the mean $m$ of the distribution of training values in a leaf. This is the value that $\phi(\b{x})$ returns if the input $\b{x}$ is routed to that leaf. So $\phi(\b{x})$ is piecewise constant in that it returns the constant value $m$ for every $\b{x}$ that lands in the box corresponding to a given leaf.

The mean $m$ of a set of values $y_1, \ldots, y_j$ minimizes the empirical variance, $$ \sigma^2 \propto \sum_{i=1}^j (y_i - m)^2\;, $$ and one often uses that variance as a measure of impurity when training a regression tree.

We make the following additional simplification in this part:

  • Only fully grown trees are considered. These are trees whose leaves contain perfectly pure training data.
  • The dimensionality of the data space $X$ is $d = 2$. In this way, we can draw the boxes at the leaves of any tree, and paint them with colors that encode the means stored in the boxes.

Of course, adjacent boxes may happen to have the same value $m$.

We saw in class and in the notes (Figure 1 (b)) how the partition $P$ can be drawn when the data space is in $\mathbb{R}^2$: First draw a box around the entire data set (a box is an axis-aligned rectangle). Then draw a line that splits that box into two new boxes according to the split information (dimension and threshold) at the root. Proceed recursively on the new boxes. You will implement this method in this part.

def shade_box(box, color):
    corner = box.left, box.bottom

def make_colormap(data):
    value_range = [np.minimum(0, np.min(data.y)), np.maximum(0, np.max(data.y))]
    half_range = max(-value_range[0], value_range[1])
    return SimpleNamespace(
        map=colormaps['RdYlBu'],
        norm=colors.CenteredNorm(vcenter=0, halfrange=half_range)
    )

The sklearn Tree¶

You may skip this section if you know what the class sklearn.tree.DecisionTreeRegressor looks like.

If h is a trained instance of the class sklearn.tree.DecisionTreeRegressor, the attribute t = h.tree_ contains a vector representation of the resulting tree. Nodes are numbered from 0 up, in an order that does not matter except for the fact that the root of the tree is node zero. The key attributes of t are as follows, where $n$ is the number of nodes in the tree.

  • t.threshold: A numpy vector of length $m$, the number of nodes in the tree, such that t.threshold[k] is the decision threshold at node k. This threshold is $\tau.t$ in the class notes.
  • t.feature: A numpy vector of length $m$ such that t.feature[k] is the decision dimension at node k. This dimension is $\tau.j$ in the class notes and is either 0 or 1 in our case.
  • t.value: A numpy array of shape (m, 1, 1) and such that t.value[k] can be passed to value_color to obtain an appropriate color for shading the box for node k.
  • t.children_left: A numpy vector of length $m$ such that t.children_left[k] is the index of the root of the left child of node k. This child is $\tau.L$ in the class notes. If a node has no left child, the index is set to a negative number.
  • t.children_right: A numpy vector of length $m$ such that t.children_right[k] is the index of the root of the right child of node k. This child is $\tau.R$ in the class notes. If a node has no right child, the index is set to a negative number.

The data structure t is further explained here if you need more detail (you shouldn't really need more than stated above for this problem). Regressor trees and classifier trees have the same structure.

Keep in mind that, given how regression trees are constructed, a node has either zero or two children.

Programming Notes¶

  • Write your function recursively. To this end, it is easier to make draw_tree a (non-recursive) function that in turn calls a sub-function, say, draw(node, box) that calls itself recursively. If draw is defined inside draw_tree, the tree t and the color_scheme are known to draw and need not be passed explicitly to it as arguments in the recursive calls. After defining draw, the function draw_tree starts the recursion by calling

      draw(0, b)

    where 0 is the index of the root and b is the box passed as the second argument to draw_tree.

  • The draw sub-function does the following for every internal node:

    • Draws the single line that splits the box b in two. Note that the endpoints of the line depend on b and t.threshold and t.feature.

    • Splits the box b into two new boxes to be passed to the left and right child of the current node.

    • Calls itself recursively to draw the subtrees with roots t.children_left[node] and t.children_right[node].

  • For a leaf with index node, the draw subfunction shades the leaf box with the appropriate color. Use the provided function value_color and shade_box to this end. The tree t stores a mean value $m$ with each leaf in t.value[node], and you can pass that value and the color_scheme to value_color to make the color to pass as the second argument to shade_box.

  • Note carefully that each leaf box is shaded with shade_box, while the boundaries of the boxes are a byproduct of drawing the splitting lines for internal nodes, as outlined above.

  • To make sure that each child gets a separate box (as opposed to a reference to the same box), make two copies of box and modify the box boundaries appropriately in the copies, without altering the original box. This is done most cleanly through a separate helper function. For example, if you want to replace box.left with a new value, you could do this by saying

      left_box = replace_side(box, 'left', value)

    where replace_side is defined as shown below.
