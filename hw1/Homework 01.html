A Qiuck Review of Regression

Regression fits a model to a data set, and is a process you are familiar with from other courses.

The data set is a set of input-output pairs, often called the training set:

$$ T = \{(x_1, y_1), \ldots, (x_n, y_n) \} \;. $$

The model is some function that takes an $x$ and produces the corresponding $y$:

$$ y = f(x, \b{p}) \;. $$

The real numbers in the vector $\b{p}$ are parameters.

An example you are familiar with is line regression, where $x$ and $y$ are the horizontal and vertical coordinate of a point on the plane. The function $f$ is

$$ y = a + b x $$

and the parameter vector is $\b{p} = (a, b)$.

The goal of regression is to find a vector $\b{p}^\ast$ of parameters so that $f$ predicts the correct values on $T$ as closely as possible:

$$ f(x_i, \b{p}^\ast) \approx y_i \;\;\;\text{for}\;\;\; i = 1, \ldots, n\;. $$

In the example, regression finds values $a^\ast$ and $b^\ast$ for the intercept $a$ and the slope $b$ of the line that passes closest to the data points in $T$.

To define the problem well, one needs to state what "closest" means, that is, to define a metric in which this approximation is measured. This metric is called the loss function

$$ \ell(y, \hat{y}) $$

where $y$ is the true result and $\hat{y}$ is what the function $f$ produces. So one measures the error in the approximation above by the loss

$$ \ell(y_i, f(x_i, \b{p}^\ast)) $$

for each data point in $T$. The loss function $\ell$ is designed so that it is large when the approximation is poor, and it goes to zero when the approximation is perfect.

A common loss function for line regression is the square of the difference:

$$ \ell(y, \hat{y}) = (y - \hat{y})^2 \;. $$

The overall error is the average loss over $T$, and is called the risk:

$$ L(\b{p}) = \frac{1}{n} \sum_{i=1}^n \ell(y_i, f(x_i, \b{p})) \;. $$

So line fitting can be stated as the following optimization problem:

$$ a^\ast, b^\ast = \arg\min_{a, b\in\reals} \frac{1}{n} \sum_{i=1}^n [y_i - (a + b x_i)]^2 \;. $$

The notation $\arg\min_{a, b\in\reals}$ means "the values of real numbers $a$ and $b$ that make the following quantity as small as possible."

You have seen in earlier courses that the solution to this problem can be computed as follows:

$$ \begin{eqnarray*} m_x &=& \frac{1}{n} \sum_{i=1}^n x_i \\ m_y &=& \frac{1}{n} \sum_{i=1}^n y_i \\ b^\ast &=& \frac{\sum_{i=1}^n (x_i - m_x) (y_i - m_y)}{\sum_{i=1}^n (x_i - m_x)^2} \\ a^\ast &=& m_y - b^\ast m_x \;. \end{eqnarray*} $$

Part 3: Basics of Voronoi Diagrams in 2D

We saw in class that any classifier partitions its domain into decision regions. For the $1$-nearest-neighbor ($1$-NN) classifier, we also saw that these regions are the cells in a Voronoi diagram. This is the case regardless of the dimensionality of the data. However, we can only draw Voronoi diagrams in two dimensions (or, with difficulty, in three).

This part of the assignment looks at some of the most the basic elements of Voronoi diagrams on the plane: The bisector of two points, the intersection of triples of bisectors, and the Voronoi diagram of three points.

The Bisector of Two Points

The decision boundary between two points $\b{a}$ and $\b{b}$ is called the bisector of $\b{a}$ and $\b{b}$, and is a key concept in Nearest-Neighbor predictors. The Voronoi diagram is made by line segments, each of which lies on one of the pairwise data-point bisectors.

While one can define distances in a variety of metrics, we always use Euclidean distance in this part. Given two points $\b{a} = (a_1, a_2)$ and $\b{b} = (b_1, b_2)$ on the plane, their distance is therefore

$$ d(\b{a}, \b{b}) = \sqrt{(a_1 - b_1)^2 + (a_2 - b_2)^2} \;. $$

Part 4: Decision Regions of NN Classifiers in 2D

Even in two dimensions, the concept of Voronoi diagram is no longer helpful when considering $k$-NN decision boundaries with $k > 1$.

Approximate Decision Regions

A simple way to display approximate decision regions for a classifier $h(\b{x})$ where $\b{x}$ is in $\reals^2$ is to make an image of classifier responses on a grid.

Specifically, create a grid of, say, $300\times 300$ two-dimensional points evenly spaced in a rectangle that bounds the data (this rectangle is specified by data['domain'] in the 2D version of the MNIST dataset retrieved above).

Assign a different color to each possible label. Run the classifier $h$ on each grid point $\b{x}$, and assign to that grid point the color corresponding to predicted label $y = h(\b{x})$. You can then use matplotlib.pyplot.imshow to display the image. Regions for different labels will have different colors, and a legend in the figure can show which colors are associated to which labels.

Thankfully, the classifiers provided in the scikit-learn package for machine learning (loaded with import sklearn) can take all the grid points at once and return all the labels at once, so no explicit for loops are needed.

This visualization method is approximate because it pixelizes the regions, but with enough grid points the approximation is often acceptable.

Keep in mind that accuracies are given as fractions of 1, so an accuracy of 0.9 is ninety percent accuracy. This is the fraction of points in either training or test set that the classifier computes correctly.

Also, we say that a predictor generalizes well if its test accuracy is not much worse than its training accuracy. So a predictor can generalize well and yet have poor performance. This would be the case, for instance, with a training accuracy of, say, 0.2 and a test accuracy of 0.23. Both are abysmal, so performance is poor, but test accuracy is better than training accuracy, so the predictor generalizes well.

The difference test accuracy $-$ training accuracy is often used as a measure of an algorithm's ability to generalize. Predictors that generalize well have positive or not very negative differences. So "generalizing well" is not a binary concept, but a matter of degree.
